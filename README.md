# 🌾 FarmTick | 智能务农系统

本项目是对 [gzchenjin2/hokfarming](https://github.com/gzchenjin2/hokfarming) 开源项目的深度重构与架构升级版本。

在完美保留原版优秀的 UI 交互、数据中心与账号管家的基础上，本项目对底层计算逻辑进行了彻底的“状态机”重构，并将原本的单文件（Big Ball of Mud）重构为**高内聚、低耦合的企业级模块化架构**，大幅提升了农场防偷菜博弈的精准度与代码的可维护性。

## 🚀 核心升级特性

### 1. 📅 纯净状态机引擎 (核心重构)
废弃了原版基于静态数学公式的预测模型，引入了底层**“1分钟步进式”时间沙盘状态机**，真实模拟作物的剩余寿命与水分消耗。
* **🌙 智能护肝模式**：支持设定个人作息时间。引擎会自动在你的梦境期间智能穿插“睡前强行补水”动作；并拥有“首日修仙豁免权”，完美契合真实人类作息。
* **⚖️ 干旱顺延算法**：精准模拟干旱期间的时间流逝，利用 `剩余总寿命扣减算法`，哪怕睡懒觉，也只会造成数学上最微小的顺延。
* **🚨 深度睡眠防坑拦截**：如果你种下作物的斩杀时间刚好落在“入睡后1h至醒来前1h”的绝对死穴，系统会弹出红色阻断警告，拒绝让你成为半夜定闹钟的怨种。

### 2. 🤫 独家防偷菜“憋气”逻辑
全网独家的“防被偷博弈算法”。
* 引擎会严密监控 10% 的浇水冷却 CD 门槛。如果在即将成熟时浇水会导致 CD 锁死（进而无法手动斩杀被偷菜），系统会**强制拦截浇水动作**。
* 在时间轴上生成 **"🤫 憋气等斩杀"** 节点，让植物干着，直到时间流逝刚好足够一刀秒熟。把主动权死死捏在自己手里！

### 3. ⏱️ 经典计算器与账号管家
* 完美保留原版“极限收割计算器”，实时给出 `建议自然成熟` 或 `终极秒熟倒计时`。
* 账号管家支持 LocalStorage 本地存储、滑动复选、CSV 备份导入导出，以及从计算器/排期器一键同步状态。

## 📁 项目架构 (企业级模块化)

为了摆脱单文件 HTML 带来的维护噩梦，本项目采用了**数据与视图分离**及**按域模块化**的现代前端架构。纯原生 HTML/JS/CSS 实现，无需 Webpack/Vite 编译，极度轻量。

```text
FarmTick/
├── index.html            # 唯一入口文件 (仅保留极简的 DOM 骨架)
├── css/
│   └── style.css         # 全局样式表 (UI 颜色、排版、响应式适配)
├── js/
│   ├── config/
│   │   └── data.js       # 核心数据抽离 (包含所有作物、等级、升级所需静态数据表)
│   ├── core/
│   │   ├── utils.js      # 全局公共工具库 (时间格式化、计算等通用方法)
│   │   └── alarm.js      # 闹钟与通知引擎 (兼容 Capacitor 原生推送与 Web 端回调)
│   ├── modules/
│   │   ├── ui.js         # 界面交互中心 (折叠卡片、Tab 切换、表格排序与渲染)
│   │   ├── calculator.js # 极限计算器模块 (单次数据的数学推演逻辑)
│   │   ├── timeline.js   # 智能排期器模块 (核心！1分钟沙盘状态机引擎)
│   │   └── account.js    # 账号管家模块 (本地存储、增删改查及 CSV 数据迁移)
│   └── app.js            # 应用启动器 (生命周期挂载、模块初始化串联)
└── README.md             # 项目说明与交接文档
